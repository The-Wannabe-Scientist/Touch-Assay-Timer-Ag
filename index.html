User Guidelines

This tool is designed to run simple touch / stimulus–response assays.

1. Structure (Assays, Trials, & Runs)

An Assay defines your experiment's parameters (e.g., interval length, number of stimuli).
Each assay contains multiple Trials, which group your tests by genotype or condition.
Each trial consists of multiple Runs (typically testing one animal per run).

2. Conducting a Run

Recording Responses: Tap the large recording button (or press Spacebar) to register a response.
Interval Scoring: Taps are recorded per interval, not by exact reaction time. Multiple taps within the same interval are treated as a single positive response.
Cues: The app uses high-precision audio to signal when to apply the stimulus. If you have the optional BLE Haptic Armband, ensure it is connected before starting.

3. Data Integrity

Ineligible Runs: If a run is stopped early or a trial is abandoned before completion, its data is marked as ineligible to prevent skewed analysis.
Local Storage: Your data is automatically saved locally to your device and is fully accessible offline.

4. Exporting & Managing Data

Excel Export: All completed trial data, including raw interval responses, touch indices, and binned percentages, can be exported to Excel for external analysis.

Important Data Warnings

Permanent Deletion: Deleting an assay permanently removes all associated trials and run data from your device. Always export your data before deleting an assay.
Local Storage: Your data is saved to this browser. Clearing your browser's site data/cache will permanently delete all your recorded data unless exported. Do not use Incognito mode.
